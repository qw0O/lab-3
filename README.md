## Технологии и алгоритмы

Проект реализован на Python с использованием библиотеки collections и класса deque, которые обеспечивают эффективную работу по принципу FIFO (First-In-First-Out) со скоростью $O(1)$ для операций добавления и удаления задач.

### Сложность операций

| Операция | Метод | Временная сложность |
| --- | --- | --- |
| Добавление (Enqueue) | `append()` | **O(1)** — константное время |
| Удаление (Dequeue) | `popleft()` | **O(1)** — константное время |
| Поиск по ID | `has_task_by_id()` | **O(n)** — линейный поиск |

## Структура кода

### Класс `Task`

Объектная оболочка для задачи. Содержит:

* `task_id`: Уникальный идентификатор.
* `description`: Описание задачи.
* `priority`: Приоритет выполнения.

### Класс `TaskQueue`

Реализация логики очереди.

* Использует инкапсулированный объект `_queue` для хранения данных.
* Включает проверку типов данных (`isinstance`) для предотвращения ошибок при добавлении некорректных объектов.
